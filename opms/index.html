MV

Сервис заполнения отчётной БД
  • Очередь событий - в БД PMS
  • Читает из очереди события
  • Раскладывает по отчётным таблицам
  • Миграции MV - flyway
  • Возможна инициализация из рабочей БД

flyway миграция


<changeSet
    author="amelnikov"
    context="update"
    id="add_natural_client_fb_identifier">
    <preConditions
        onFail="MARK_RAN"
        onFailMessage="MV_NATURAL_CLIENT.facebookId already exists">
        <not>
            <columnExists
                columnName="facebookId"
                tableName="MV_NATURAL_CLIENT"/>
        </not>
    </preConditions>
    <addColumn tableName="MV_NATURAL_CLIENT">
        <column
            name="facebookId"
            type="${type.string}"/>
    </addColumn>
</changeSet>

Slave

  • “Урезанный” PMS, который строит отчёт
  • Связь PMS-Slave через Akka
  • Получает задание, выполняет, возвращает xml
  • Задания: Построение, отрисовка, выгрузка
  • Если Slave не вынесен, то всё - внутри PMS
  • Но общение всё равно через Akka
